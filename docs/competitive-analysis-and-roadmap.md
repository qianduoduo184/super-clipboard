## 1. 当前能力快照（v1.0.7）

| 维度 | 现状 |
|------|------|
| 捕获 | 文本、HTML、图片、文件列表 |
| 写回粘贴 | **四类均已实现**：文本、HTML（当前强制转纯文本）、图片（`CF_DIB`）、文件列表（`CF_HDROP`）。仅 `#[cfg(windows)]` 写回路径尚未真机验证 |
| 存储 | SQLite（WAL）、FTS5 trigram 搜索、按 hash 去重、收藏、置顶、软删除、最大条数(默认 10k)/保留天数(默认 30d)、自定义数据/日志目录 |
| 检索 | 关键词搜索（含 CJK 子串）、类型过滤（全部/收藏/文本/图片/文件）、可配置导航过滤器 |
| 交互 | 全局快捷键 `Ctrl+Shift+V`、快速面板、键盘导航（方向键/回车/删除/ESC）、拖拽排序、右键菜单、失焦自动隐藏、智能 ESC、可拖动预览分栏 |
| 系统 | 托盘、开机自启、自动更新、明暗主题、备份导入/导出 |
| 平台 | 仅 Windows |

## 2. 竞品横向对比

选取覆盖面最广、口碑最好的几款作参照：

- **Ditto**（Windows / 开源）：老牌，数字键快速粘贴、纯文本粘贴、剪贴板分组、加密、局域网多机同步、贴近光标定位、合并多条粘贴、条目内编辑。
- **CopyQ**（跨平台 / 开源）：标签页(tab)分组、脚本与自动化、按内容正则触发动作(command)、条目可编辑、纯文本粘贴、加密标签页、标签(tag)、拖放。
- **Windows 剪贴板历史 Win+V**（系统内置）：置顶、Emoji/GIF、跨设备云同步、纯文本粘贴、**应用可声明排除敏感内容**。
- **Maccy / Paste（macOS）**：极致键盘化、快速过滤、纯文本粘贴、收藏板(boards)。
- **PasteBar / Paste（商业）**：收藏板/集合、带变量的代码片段(snippet/模板)、丰富组织能力、快捷键。
- **ClipboardFusion**：格式清洗宏、内容触发器、同步。

## 3. 缺陷与不足（对照竞品）

按"是否阻碍作为剪贴板管理器的核心价值"分层。

### 3.1 🔴 核心缺陷（几乎所有竞品都有，本应用缺失，直接削弱可用性）

1. **HTML 粘贴丢失格式，且无"以纯文本粘贴"选项**
   - 现状：`copy_item` 对 HTML 条目一律 `html_to_plain_text` 转纯文本（`commands.rs:92-96`）。粘贴富文本必丢样式，也无法选择保留。
   - 竞品：Ditto/CopyQ/Win+V 默认保留 HTML 格式，并另提供"以纯文本粘贴"。
   - 借鉴方案：①新增 `CF_HTML` 写回分支保留格式；②详情区与右键菜单加显式"以纯文本粘贴"，复用现有纯文本路径。

2. **无法排除敏感来源 / 不记录密码类内容**
   - 现状：任何复制都会入库，包括密码管理器复制的密码、验证码、银行卡号，明文落 SQLite。既是隐私也是安全短板。
   - 竞品：Win+V 尊重应用声明的排除格式；多数管理器提供"应用排除名单"。
   - 借鉴方案（隐私 + 安全，优先级高）：
     - 尊重 Windows 剪贴板排除格式：`ExcludeClipboardContentFromMonitorProcessing`、`CanIncludeInClipboardHistory`（DWORD=0）（KeePass/1Password 等会设置；**实现时以 Microsoft 官方剪贴板历史格式文档为准核对格式名**）。检测到即跳过入库。
     - 用户可配置的"应用排除名单"（按前台进程名/exe 排除）。
     - 可选：敏感内容启发式识别（疑似密码/卡号）→ 不记录或标记为敏感、自动过期。

> **已具备（2026-07-03 复核纠正）**：图片 / 文件粘贴写回**已在代码中实现**（`copy_item` 分支 + `win.rs` 的 `CF_DIB`/`CF_HDROP` 写入），本文早前基于过时的 CLAUDE.md 误列为缺陷，现更正。仅剩真机验证未做。

### 3.2 🟡 高价值增强（竞品普遍具备，能显著提升效率）

4. **数字键 / 快捷键快速粘贴第 N 条**（Ditto 招牌）：面板打开后按 `1~9` 直接粘贴对应条目。纯键盘流的核心加速器。

5. **片段 / 模板 / 收藏板（分组组织）**（CopyQ tab、PasteBar boards、Paste）：把常用内容固定成可命名的组或模板（支持变量占位）。当前只有扁平的"收藏/置顶"，缺少分组与常驻片段库。

6. **条目内编辑后再粘贴**（Ditto/CopyQ）：粘贴前微调文本（改错别字、删多余空行）。

7. **合并多条粘贴 / 粘贴堆栈**（Ditto）：多选若干条按顺序合并为一次粘贴；或"连续粘贴"模式（粘贴后自动指向下一条）。

8. **内容感知快捷动作**（CopyQ command / ClipboardFusion trigger）：识别 URL/邮箱/颜色值/文件路径/代码，提供"打开链接、生成二维码、颜色预览、以代码高亮查看"等一键动作。（与 TODO 既有「智能分类」「快捷操作」合并深化。）

9. **贴近光标 / 上次位置的面板定位**（Ditto）：面板出现在鼠标或插入符附近，减少视线跳转。当前为固定窗口位置。

10. **多选 + 批量操作**：批量删除/收藏/导出，配合合并粘贴。

### 3.3 🟢 长期 / 差异化（部分已在 TODO 长期规划）

12. **端到端加密存储**：已在 TODO「剪贴板加密」。可先做"敏感条目单独加密"再考虑整库。
13. **更多格式**：RTF（富文本粘贴到 Word/邮件保留样式）、颜色、Emoji/GIF 面板。已在 TODO「更多格式」。

## 4. 建议优先级与路线

以"投入产出比 + 是否补齐核心短板"排序：

| 优先级 | 功能 | 理由 |
|--------|------|------|
| ~~P0~~ ✅ | ~~图片 / 文件粘贴写回~~ | 复核发现**已实现**，仅剩真机验证 |
| P0 | 富文本 HTML 粘贴 + 纯文本开关（3.1-1） | 高频，当前 HTML 必丢样式 |
| P0 | 敏感来源排除 + 应用排除名单（3.1-2） | 隐私/安全短板，避免明文存密码 |
| P1 | 数字键快速粘贴（3.2-4） | 键盘流核心加速，改动集中在前端 |
| P1 | 片段/模板/分组（3.2-5） | 组织能力质变，需 schema 扩展 |
| P1 | 内容感知快捷动作（3.2-8） | 与既有 TODO 合并 |
| P2 | 条目内编辑（3.2-6）、合并粘贴（3.2-7）、面板贴近光标（3.2-9）、多选批量（3.2-10） | 效率增强 |
| P3 | 局域网同步、加密、RTF、OCR、检索增强、首次引导（3.3） | 长期/差异化 |

> 说明：本表为建议排序，实际排期以用户确认为准。已存在于 TODO 长期规划的项（云同步/加密/更多格式/OCR/智能分类/快捷操作）在此做了对标与拆分，落 TODO 时保持引用不重复。
