## 测试目标
验证历史记录项和导航过滤器的拖放排序功能正常工作。

- `src/styles.css` (行 229-310)

### 2. 导航过滤器拖动
**修改：**
- 添加 `dataTransfer.effectAllowed` 和 `dropEffect`

- 添加悬停状态和过渡动画

**文件：**
- `src/App.tsx` (行 632-665)

## 测试步骤

### ✅ 测试 1：历史记录项拖动排序

1. **启动应用**
   ```bash
   npm run tauri dev
   ```

2. **准备测试数据**
   - 确保有至少 5 个剪贴板历史记录
   - 如果没有，复制一些文本创建历史记录

3. **测试拖动**
   - 将鼠标悬停在历史项上，光标应显示为 **抓手图标** (grab)
   - 点击并按住历史项，光标应变为 **抓握图标** (grabbing)
   - 拖动项时，应该看到：
     - 被拖动项半透明 (opacity: 0.4)
     - 虚线边框 (dashed border)
     - 背景变为主色调半透明
   - 拖动到目标位置并释放
   - **预期结果：** 项目顺序改变，显示"排序已保存"

4. **验证持久化**
   - 关闭并重新打开应用
   - **预期结果：** 排序保持不变

5. **测试边界情况**
   - 拖动到第一项位置
   - 拖动到最后一项位置
   - 拖动相邻项
   - **预期结果：** 所有情况都正确重排

### ✅ 测试 2：导航过滤器拖动排序

1. **打开导航配置**
   - 点击过滤器行右侧的 **滑块图标** (⚙️)

2. **测试拖动**

     - 光标应显示为 **移动图标** (move)
     - 背景应变为高亮
   - 拖动一个过滤器：
     - 被拖动项半透明 + 虚线边框 + 轻微缩小
     - 拖动到目标位置并释放
   - **预期结果：** 过滤器顺序改变

3. **验证应用**
   - 关闭配置面板
   - **预期结果：** 顶部过滤器按钮顺序与配置一致

4. **验证持久化**
   - 刷新应用或重启
   - 打开导航配置
   - **预期结果：** 排序保持不变

## 已知限制

1. **虚拟滚动**：只能拖动可见区域内的项目
2. **"全部"过滤器**：故意禁用拖动，因为它必须在第一位
3. **触摸设备**：需要原生拖放支持（部分移动浏览器不支持）

## 故障排除

### 拖动不工作
1. 检查 `draggable={true}` 属性是否存在
2. 检查浏览器控制台是否有 JS 错误
3. 确认 `isDraggingDisabled` 为 `false`

### 拖动但不重排
1. 检查 `handleDropItem` 或 `handleNavFilterReorder` 是否被调用
2. 验证后端 `reorder_items` 命令是否成功
3. 检查 `reorderItemsByDrag` 逻辑

### 视觉反馈缺失
1. 检查 `.dragging` CSS 类是否应用
2. 确认 `draggingId` 或 `draggingFilterKey` 状态更新
3. 查看浏览器开发者工具的样式面板
